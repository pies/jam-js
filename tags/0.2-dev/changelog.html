JAM
A Javascript library that aims to impress.

JAM 0.2-dev released

Changelog:

  • (new) Number.prototype.round() (i.e. var three = (10/3).round()),
  • (new) Number.prototype.abs() (i.e. var two = (-2).abs()),
  • (new) Number.prototype.or() (i.e. var five = (0).or(5);),
  • (new) added a set of IE 5.0 compatibility fixes by Dean Edwards, although I'm not sure if it helps JAM work with IE5 in any way, to be decided if it stays or goes,
  • (new) classes extended with extend() get an additional method -- bound(), i.e. connect(obj, 'click', MyClass.bound('Foo')) is the same as connect(obj, 'click', MyClass.Foo.bind(MyClass)) but shorter
  • (new) JAM.Event tracks mouse button events to be able to provide correct information about currently pressed mouse buttons at any point of execution, available via the event.mouse.buttons array,
  • (new) String.prototype.reverse (i.e. 'abc'.reverse() == 'cba'),
  • (new) Array.prototype.add() is same as Array.push(), but chainable (i.e. [a,b].add(c) returns [a,b,c] whereas [a,b].push(c) does not),
  • (new) Array.prototype.only() limits contents of the array to elements present in the provided array (a logical AND on two arrays, i.e. [a,b,c].only([a,c,e]) == [a,c]),
  • (new) Array.prototype.not() removes elements present in the provided array (a logical NOT, i.e. [a,b,c].not([a,c,e]) == [b]),
  • (new) Array.prototype.empty() which simply empties the array, but is chainable
  • (new) Element.prototype.name() returns element's tagName or 'element' of tagName not present
  • (new) Element.prototype.set() called with just one, non-array parameter (i.e. $('#bar').set('Hello')) sets element's value (if applicable) or innerHTML (otherwise); likewise, element.get() called with no parameters returns element.value (if applicable) or element.innerHTML (otherwise)
  • (new) Element.prototype.placeOver() tries to place elements centered h and v over an element
  • (new) JAM.Dom.selector tries to be compatibile with Prototype's $() function (i.e. $('foobar') tries to find all <foobar> tags, and if it finds none such, tries $('#foobar') automatically),
  • (new) JAM.Dom.Draggable (applied with JAM.Dom.Draggable.apply(element)) allows to drag an element around with mouse, see the code for details
  • (new) JAM.Dom.Resizable (applied with JAM.Dom.Resizable.apply(element)) allows to resize an element around with mouse, see the code for details
  • (update) refactored the main JAM class
  • (update) some Element.prototype refactoring and changes, notably getStyle(), setStyle() and undoStyle() renamed to getCss(), setCss() and restoreCss(), and position() (via getPosition()) returns element's cumulative position which can break some code,
  • (fixed) isObject() alias didn't get exported
  • (fixed) packed code had errors because of a few missing semicolons,
  • (fixed) Element.off(N) didn't disconnect the event binding, now it disconnects N last bindings (or all if no N),

Packed filesize still below 20kB and there's room for improvements and deletions. That was a busy two weeks, let me tell you.
